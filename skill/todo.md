# ToDo 管理

## tools

- `todo_list`
    - ToDoリストを一覧表示する
    - 引数 `include_done` (bool, 省略可能)
        - `false`（デフォルト）: 未完了のタスクだけ返す
        - `true`: 完了済みも含めて全件返す
    - 返り値: `{ status, count, todos: [ { id, title, note, done, created_at, done_at? } ] }`
- `todo_add`
    - 新しいタスクを追加する
    - 引数 `title` (string, 必須): タスクのタイトル
    - 引数 `note` (string, 省略可能): 補足メモ
    - 返り値: 追加したタスクのオブジェクト
- `todo_done`
    - 指定したIDのタスクを完了にする
    - 引数 `id` (integer, 必須): 完了にするタスクのID
    - 返り値: 完了後のタスクのオブジェクト
- `todo_delete`
    - 指定したIDのタスクを削除する
    - 引数 `id` (integer, 必須): 削除するタスクのID
    - 返り値: 削除したタスクのID

## スキルの手順

### タスクの追加
1. ユーザーが「〇〇をToDoに追加して」「〇〇をしないといけないの覚えておいて」などと言ったら
2. `todo_add` を呼び出し、`title` にタスク名を渡す
3. メモや補足があれば `note` にも渡す
4. 追加後、「〇〇を追加しました（ID: N）」と報告してスキルを終了する

### タスクの一覧表示
1. ユーザーが「ToDoを見せて」「今日は何したらいい？」などと言ったら
2. `todo_list` を呼び出す（通常は `include_done` は省略してOK）
3. 取得した todos を箇条書きで「ID: N - タイトル」の形式でわかりやすく表示する
4. 件数が0件なら「ToDoは今のところ何もないですよ」と伝える
5. 「完了済みも見たい」と言われたら `include_done: true` で再度呼び出す

### タスクの完了
1. ユーザーが「〇〇が終わった」「〇〇を完了にして」などと言ったら
2. タスクのIDが不明な場合は `todo_list` で一覧を取得してIDを特定する
3. `todo_done` を呼び出し、`id` を渡す
4. 完了後、「ID N の〇〇を完了にしました」と報告してスキルを終了する

### タスクの削除
1. ユーザーが「〇〇を消して」「〇〇をリストから削除して」などと言ったら
2. タスクのIDが不明な場合は `todo_list` で一覧を取得してIDを特定する
3. `todo_delete` を呼び出し、`id` を渡す
4. 削除後、「ID N を削除しました」と報告してスキルを終了する

## 注意事項

- タスクのIDは一覧で確認してから操作すること
- 「完了」と「削除」は別の操作。完了は記録として残り、削除は完全に消える
- タイトルが曖昧で複数候補がある場合は、ユーザーに確認してから操作すること
